Rutas (Routing) en Angular

El **Sistema de Rutas (Routing)** en Angular permite definir cómo los usuarios navegan entre diferentes vistas o componentes dentro de la aplicación sin recargar la página completa. Esto crea una **experiencia de usuario fluida y dinámica**, esencial en aplicaciones modernas de una sola página (SPA).

Las rutas se definen en un array de objetos `Routes`, generalmente en un módulo de enrutamiento separado (por convención, `app-routing.module.ts`).

Ejemplo de Configuración de Rutas (app-routing.module.ts):

<!-- Importa los módulos necesarios de Angular Router. --> import { NgModule } from '@angular/core'; import { RouterModule, Routes } from '@angular/router'; <!-- Importa los componentes que serán asociados a las rutas. --> <!-- Cada componente representa una "vista" de nuestra aplicación. --> import { HomeComponent } from './components/home/home.component'; import { ComponentExplanationComponent } from './components/component-explanation/component-explanation.component'; import { ServiceExplanationComponent } from './components/service-explanation/service-explanation.component'; <!-- ... y otros componentes de tu aplicación --> <!-- Define el array de rutas. Cada objeto de tipo Route define una ruta individual. --> const routes: Routes = [ <!-- Ruta raíz: Cuando la URL es '/', Angular cargará el HomeComponent. --> <!-- 'pathMatch: 'full'' asegura que la URL coincida exactamente con la cadena vacía. --> { path: '', component: HomeComponent, pathMatch: 'full' }, <!-- Ruta para el componente de explicación de Componentes. --> <!-- Cuando la URL es '/componentes', se cargará ComponentExplanationComponent. --> { path: 'componentes', component: ComponentExplanationComponent }, <!-- Ruta para el componente de explicación de Servicios. --> { path: 'servicios', component: ServiceExplanationComponent }, <!-- Puedes añadir más rutas aquí para otros componentes y funcionalidades. --> <!-- Ejemplo de ruta con un parámetro: --> <!-- { path: 'producto/:id', component: ProductDetailComponent }, --> <!-- Ruta comodín (**): Captura cualquier URL que no haya coincidido con las rutas anteriores. --> <!-- Es útil para implementar una página "404 No Encontrado" o, como en este caso, --> <!-- redirigir al usuario a la página de inicio para URLs no válidas. --> { path: '**', redirectTo: '' } ]; @NgModule({ <!-- RouterModule.forRoot(routes) configura el enrutador a nivel de la aplicación raíz. --> <!-- Esto debe hacerse solo una vez en el módulo principal de enrutamiento (generalmente AppModule o AppRoutingModule). --> imports: [RouterModule.forRoot(routes)], <!-- Exporta RouterModule para que las directivas de enrutamiento (como routerLink y router-outlet) --> <!-- estén disponibles para otros módulos que importen este AppRoutingModule. --> exports: [RouterModule] }) export class AppRoutingModule { }

En tus plantillas HTML, usas la directiva **`routerLink`** para crear enlaces de navegación y **`<router-outlet>`** para indicar dónde debe renderizarse el componente correspondiente a la ruta activa.

<!-- app.component.html (ejemplo de uso) --> <nav> <a routerLink="/">Inicio</a> <a routerLink="/componentes">Ver Componentes</a> <a routerLink="/servicios">Ver Servicios</a> </nav> <!-- El <router-outlet> es el placeholder donde Angular inserta el componente de la ruta activa --> <router-outlet></router-outlet>
